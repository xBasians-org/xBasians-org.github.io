В Visual FoxPro нет встроенной многопоточности, как в C++ или C#. Но есть обходные пути:

1. Использование COM-серверов

Можно запускать несколько экземпляров FoxPro COM-объектов, каждый из которых работает независимо.


* MathServer.prg
DEFINE CLASS MathServer AS Session OLEPUBLIC
    FUNCTION Multiply(x, y)
        RETURN x * y
    ENDFUNC
ENDDEFINE

* Компиляция:
* BUILD MTDLL MathServer.dll FROM MathServer.prg

2. Запуск нескольких процессов FoxPro

FoxPro-приложения можно запускать как отдельные процессы, обмениваясь данными через:

  • файлы (DBF, XML, текстовые логи)
  • OLE/COM-объекты
  • сокеты (через Winsock ActiveX)

3. Эмуляция потоков через таймеры

В приложениях VFP можно использовать Timer, чтобы имитировать асинхронное выполнение задач:


oTimer = CREATEOBJECT("Timer")
oTimer.Interval = 1000
oTimer.Enabled = .T.

DEFINE CLASS Timer AS Timer
    PROCEDURE Timer
        ? "Работаем как в отдельном потоке"
    ENDPROC
ENDDEFINE

Это не настоящие потоки, но позволяет выполнять периодические задачи без блокировки интерфейса.
